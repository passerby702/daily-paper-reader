## 摘要
扩散变压器（DiT）在生成扩散领域已被广泛采用，通过注意力和前馈（FFN）层推进查询令牌的去噪。FFN实际上充当解码视觉内容的键-值词汇表，其中值嵌入视觉语义知识。我们提出，关注对应于更复杂细节的关键查询令牌，并鼓励模型改进这些令牌，对于精细视觉生成至关重要。为此，我们提出了FocusDiT，它应用了一种屏蔽方案，专注于仅输入到FFN的关键查询令牌。屏蔽的查询可以从FFN词汇表中检索视觉令牌，并使用它们解码其视觉细节。大量的文本到图像实验验证了令牌屏蔽在增强生成性能方面的有效性。

## Abstract
Diffusion transformer (DiT) has been widely adopted in the generative diffusion field, advancing the denoising of query tokens through attention and Feed-Forward (\text{FFN}) layers. FFN actually acts as the key-value vocabulary for decoding visual contents where the value embeds the visual semantical knowledge. We present that focusing on critical query tokens corresponding to more complex details and encouraging the model to improve these tokens is essential for fine-grained visual generation. To this end, we propose FocusDiT, which applies a Masking scheme to focus on critical query tokens that are exclusively fed into FFN. The masked queries can retrieve visual tokens from the FFN vocabularies, and use them to decode their visual details. Extensive text-to-image experiments validate the effectiveness of token masking in enhancing generative performance.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：在扩散变压器（DiT）中，前馈网络（FFN）被视为一个存储视觉语义知识的关键-值词汇表，但现有方法对所有查询令牌（query token）不加区分地处理，导致那些对应复杂细节的关键令牌无法充分利用FFN词汇，限制了生成图像的细粒度和质量。
- **背景**：已有研究表明，Transformer中的FFN层相当于一个键-值记忆库，存储了从训练数据中学到的视觉概念。但在DiT中，FFN词汇表利用不充分，许多词汇条目很少被激活，且查询令牌间的激活模式差异极小，未能区分关键与非关键令牌。此外，不同层需要的词汇规模不同，手动设计层间词汇调度较为脆弱。
- **研究动机**：通过识别并聚焦于关键查询令牌，引导模型优先利用FFN词汇解码这些令牌的视觉细节，从而提升细粒度图像生成的质量和效率。

## 2. 方法论：核心思想、关键技术细节与流程

- **核心思想**：对查询令牌应用软掩码（Query Token Masking），使关键令牌（对应复杂结构）获得更大的FFN更新贡献，非关键令牌的FFN更新被抑制，从而让模型更有效地利用FFN词汇表解码细节。
- **关键技术细节**
  - **查询掩码生成网络（Q-MaskGen）**：以当前块输入特征、前一块的掩码和扩散时间步为输入，通过MLP预测每个查询令牌的掩码值（0到1之间）。时间步的引入使得不同去噪阶段能自适应调整关注区域：高噪声阶段无需强区分，低噪声阶段关键令牌（如边缘、纹理）获得更高掩码值。
  - **掩码应用**：掩码乘以FFN输出（在残差连接前），公式为 \( Z_t^{fo} = Z_t^{fi} + M_Q \cdot \text{FFN}(Z_t^{fi}) \)。反向传播时，掩码也作用于梯度，使关键令牌对应的词汇权重更新更充分。
  - **词汇重分配（Vocabulary Redistribution, VR）**：每个FFN块分为局部专家（层特定，大小 \(C_1\)）和共享专家（跨层复用，大小 \(C_2\)），保持FFN总隐藏维度 \(C = C_1 + C_2\) 不变，减少冗余参数。经消融选择共享专家 \(C_2 = 3d\) 为最优。
- **算法流程**：每个FocusDiT块依次执行自注意力、交叉注意力、查询掩码生成、FFN（通过掩码加权更新）、残差连接。掩码在训练时是软掩码，推理时可阈值化为硬掩码以跳过部分FFN层加速。
